OBJECTIU A — Fix Purfview .exe (exit 3221226505 + SRT en ubicació inesperada)
────────────────────────────────────────────────────────────────────────────────

Fitxer modificat:
  backend_nest_mvp/src/modules/projects/transcription.processor.ts

Problema:
  - faster-whisper-xxl.exe genera el SRT correctament però surt amb exit code
    3221226505 (0xC0000409 = STATUS_STACK_BUFFER_OVERRUN), que és un crash del
    CRT de Windows durant el cleanup de la pila. L'exe JA HA ACABAT la feina.
  - El SRT s'escrivia al directori de l'exe, no al directori del vídeo.
  - La implementació anterior tractava qualsevol exit != 0 com a error fatal.

Solució implementada:
  1. Constant EXIT_CODE_SECURITY_CRASH = 3221226505 (0xC0000409) identificada.
  2. Exit != 0 NO és fatal: primer busca el SRT en múltiples ubicacions:
       - path.join(exeDir, baseName + '.srt')    ← on l'exe realment escriu
       - path.join(mediaDir, baseName + '.srt')  ← prop del vídeo (fallback)
       - path.join(outDir, baseName + '.srt')    ← backend outDir (fallback)
  3. Si el SRT existeix i té mida > 0 → èxit (fins i tot amb exit code != 0).
  4. Logs diferenciats: "crash on exit detectat" vs "exit no-zero genèric".
  5. SRT copiat a outDir per consistència amb el pipeline Python.
  6. Únic error real: cap SRT trobat en cap de les 3 ubicacions.

Cap engine afectat: whisperx, faster-whisper normal, script-align → NO tocats.

OBJECTIU C — Millores LLM (speaker contamination + rebalancing local)
────────────────────────────────────────────────────────────────────────────────

Fitxer modificat:
  WhisperX_Sonilab_01-main/src/transcript_corrector.py

Problema 1 — Speaker contamination:
  LLM inseria noms de personatge al text del subtítol:
  "RUFFY: Haki busoshoku!" en lloc de "Haki busoshoku!"
  Causa: guion_block mostrava "RUFFY: text" → LLM mimificava el format.

Solució:
  a) Canviat format del guion_block de `speaker: text` a `[speaker] text`
     → desambigua el format per al LLM
  b) Afegit regla explícita al prompt:
     "5. DO NOT include speaker names in subtitle text — only the dialogue words"
  c) Afegida funció _strip_speaker_contamination(text, known_speakers):
     - Patrons de strip per known_speakers: [NAME], *NAME*, NAME:, NAME -
     - Heurística general: mot ≥2 caràcters en CAPS seguit de ':' a l'inici
     - S'aplica a cada línia del resultat LLM
  d) known_speakers extret del guio_cues per al post-processat

Problema 2 — Rebalancing local:
  Whisper de vegades barreja diàleg de personatges adjacents en un sol subtítol.
  El LLM no podia redistribuir entre subtítols veïns.

Solució:
  Afegida regla al prompt:
  "6. If Whisper has mixed/merged dialogue from different speakers in adjacent
     subtitles, you MAY redistribute text between 2-3 consecutive subtitles to
     better match the script. But keep the total count at exactly N."

Verificació mode take-llm (sense silent fuzzy fallback):
  - _correct_take_with_llm() NO té fallback a fuzzy
  - Si LLM falla → excepció propagada (registrada al log)
  - L'engine `take-llm` al backend passa llmMode='off' per configuració del modal
    → TODO pendent: el modal pot passar llmMode en futures versions
