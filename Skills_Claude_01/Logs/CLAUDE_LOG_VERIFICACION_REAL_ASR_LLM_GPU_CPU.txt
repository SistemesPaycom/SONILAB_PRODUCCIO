5. RUTA REAL D'EXECUCIÓ — LLM (mode "IA per TAKE")
================================================================================

DESCOBERTA CRÍTICA: LLM tenia FALLBACK SILENCIÓS

Flux anterior (INCORRECTE):
  1. Usuari selecciona mode "🤖 IA per TAKE"
  2. Backend crida transcript_corrector.py --method take-llm
  3. align_and_correct_by_take() crida _correct_take_with_llm() per cada TAKE
  4. _correct_take_with_llm() crida _call_ollama()
  5. Si Ollama NO està en marxa:
     except Exception: return None  ← SILENCI TOTAL, cap error
  6. llm_result = {} (buit)
  7. 0 canvis aplicats — IDÈNTIC a no fer res
  8. Usuari no veu cap indicació de fallida

Flux CORREGIT:
  1. Usuari selecciona mode "🤖 IA per TAKE"
  2. Backend crida transcript_corrector.py --method take-llm
  3. correct_transcript() fa CHECK EXPLÍCIT d'Ollama ABANS:
       ollama_ok, ollama_msg = _check_ollama_available(llm_model)
       Si NOT ok → raise RuntimeError amb missatge clar
  4. Si Ollama disponible: align_and_correct_by_take() amb verbose=True
  5. Cada crida LLM s'imprimeix a stderr: "[LLM-TAKE] TAKE #N: ... → enviant al LLM..."
  6. Resposta rebuda: "[LLM-TAKE] TAKE #N: X canvis detectats pel LLM"
  7. Si TOTES les crides fallen: raise RuntimeError (mai silenci)
  8. Backend captura l'error i el retorna al frontend com a {"error": "..."}

Endpoint de health check Ollama: http://127.0.0.1:11434/api/tags

9. LIMITACIONS QUE RESTEN
================================================================================

1. L'exe faster-whisper-xxl.exe de Purfview NO s'invoca directament.
   La implementació usa el paquet Python faster-whisper (equivalent funcional).
   Per a invocació exacta de l'exe caldria:
     - Tenir l'exe instal·lat localment
     - Afegir un engine "purfview-exe" al pipeline que el cridi com a subprocess

2. El mode take-llm requereix Ollama instal·lat i el model descarregat.
   Sense Ollama, el mode falla amb un error clar (no fallback silenciós).

3. El corrector LLM (mode fuzzy + llm_mode=fast|smart) segueix tenint possible
   fallback silenciós si Ollama no és accessible durant la correcció d'un segment
   concret. En mode take-llm el fallback silenciós ha estat eliminat completament.

4. Els paràmetres VAD de l'exe Purfview no son públicament documentats al 100%.
   Els valors usats (min_silence=300ms, speech_pad=300ms) s'aproximen a les
   observacions de la comunitat però poden diferir de l'exe exacte.
