5. COM S'HA RESTRINGIT EL LLM A ACTUAR COM ÀRBITRE LOCAL
──────────────────────────────────────────────────────────
NOVA FUNCIÓ `_llm_arbitrate_local(model, seg, candidates, prev_seg, next_seg)`:

  - El LLM ONLY rep:
    * El text ASR del segment actual
    * El text del segment anterior (context)
    * El text del segment següent (context)
    * 2-4 candidats del guió (ja filtrats per TAKE i posició)
  - El LLM ONLY pot triar una de 4 accions:
    * "no_change"           → deixar el text ASR tal qual
    * "replace_existing"    → substituir per el candidat N
    * "rebalance_with_prev" → redistribuir text amb el seg anterior
    * "rebalance_with_next" → redistribuir text amb el seg següent
  - Resposta estructurada JSON: {"action": "...", "candidate": N, "text": "..."}
  - Si la resposta no és parseable → "no_change" (fail-safe)
  - Timeout: 30s per crida (no 120s com l'anterior)

  - Diferències amb l'anterior `_correct_take_with_llm`:
    * ANTERIOR: enviava TOT el TAKE (tots els segs + totes les cues) → LLM lliure
    * NOU: envia 1 segment + 2-4 candidats → LLM arbitre local

  CONDICIONS per cridar el LLM:
    * Fuzzy score en zona ambigua: [FAST_LO=0.225, THRESHOLD=0.45)
    * Score massa baix (<0.225): no_change directe (sense LLM)
    * Score alt (>=0.45): fuzzy_replace directe (sense LLM)

7. COM ES DETECTA UNA LÍNIA REALMENT FALTANT
─────────────────────────────────────────────
NOVA FUNCIÓ `_detect_proposed_insertions(take_cues, take_segs, used_indices, take_end)`:

  Condicions ESTRICTES (totes han de complir-se):
  1. La cue NO ha estat matchejada (no és a used_local_indices)
  2. content_class és "dialogue" o "mixed" (no gesture_only, no insert_or_title)
  3. Té >= 3 paraules de text verbal real
  4. Cap segment del TAKE té similitud > 0.40 amb el text de la cue
     (si ja hi ha cobertura parcial → no proposa)
  5. Temporalment dins els límits del TAKE

  Resultat: ChangeRecord amb:
    * seg_idx = -1
    * action = "propose_new_cue"
    * start/end = estimació temporal (abs_time + 3s)
    * proposed_after_seg_idx = últim seg_idx amb start_s <= cue.abs_time

10. LIMITACIONS QUE ENCARA EXISTEIXEN
───────────────────────────────────────
A. `_correct_take_with_llm` (funció legacy) encara existeix i s'usa si es cridat
   directament. En el nouveau flux `align_and_correct_by_take` ja NO es crida
   (s'usa `_llm_arbitrate_local`). Podria eliminar-se en futura sessió.

B. El rebalancing (rebalance_with_prev / rebalance_with_next):
   - El `text_neighbor_map` acumula el text del veí però pot ser sobreescrit
     si un seg rep rebalancing de dos costats (cas poc probable però possible).
   - Si el LLM proposa un rebalancing però no proporciona `text_neighbor`,
     el text del veí no es modifica (el del seg actual sí).

C. El THRESHOLD del mode take-llm és fix (_THRESHOLD=0.45, _FAST_LO=0.225).
   En el mode fuzzy estàndard, el threshold és configurable per CLI.
   Podria unificar-se en una propera iteració.

D. Els timecodes de propose_new_cue són estimats (abs_time + 3s).
   No es calcula la durada real basant-se en els gaps entre segments.

E. La detecció de gesture_only en línies "mixtes" pot no ser perfecta
   si els tokens de gesticulació no estan al patró _GESTURE_TOKENS.
   Exemple: "(Remuga)" → no seria detectat com gesture si "Remuga" no és al patró.

F. El mode `align_and_correct` (fuzzy sense LLM) té el lookback corregit a 1,
   però el window continua sent 8 (mirant fins a 8 cues endavant). Podria
   reduir-se a 4-5 si es detecten salts inadequats.
