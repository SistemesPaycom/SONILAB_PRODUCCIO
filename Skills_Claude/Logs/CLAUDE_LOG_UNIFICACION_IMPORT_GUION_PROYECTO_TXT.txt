── ① scriptImportPipeline.ts (FITXER NOU) ───────────────────────────────────
Ruta: frontend/utils/Import/scriptImportPipeline.ts

PROPÒSIT:
  Helper compartit que encapsula tota la lògica de conversió DOCX/PDF →
  TXT canònic estructurat de SONILAB, usant exactament la mateixa ruta
  que LibraryView.tsx (el pipeline d'importació convencional).

INTERFÍCIE EXPORTADA:
  interface ImportedStructuredScript {
    fileName: string;    // nom base + ".txt"  (ex: "guio_peli.txt")
    baseName: string;    // nom base sense extensió
    content: string;     // TXT canònic (font de veritat per a MONO i parser)
    csvContent: string;  // CSV derivat (TAKE | PERSONATGE | TEXT)
    sourceType: 'slsf';  // tipus intern per a detecció legacy
  }

  async function importStructuredScriptFromFile(
    file: File,
    options?: ImportOptions
  ): Promise<ImportedStructuredScript>

LÒGICA INTERNA:
  1. Detecta extensió (.docx → importDocxFile, .pdf → importPdfFile)
  2. Aplica opcions de normalització (cleanSpaces + applyTabHeuristic per defecte)
  3. Obté el contingut TXT canònic estructurat
  4. Deriva el CSV: parseScript(content) → scriptToCsv(takes)
  5. Retorna el resultat amb totes les dades necessàries
  6. Llança Error si l'extensió no és .docx ni .pdf (amb missatge explicatiu)

IMPORTS INTERNS:
  - importDocxFile      (./docxImporter)
  - importPdfFile       (./pdfImporter)
  - ImportOptions       (./importShared)
  - parseScript         (../EditorDeGuions/scriptParser)
  - scriptToCsv         (../EditorDeGuions/csvConverter)
