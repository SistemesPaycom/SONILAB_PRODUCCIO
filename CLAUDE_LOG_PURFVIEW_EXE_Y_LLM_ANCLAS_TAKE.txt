4. OBJECTIU B — PARSING ÀNCORES TAKE (script_txt_parser.py)
================================================================================

CONTEXT DEL FORMAT:
  Les línies de diàleg SONILAB TXT segueixen el patró:
    *PERSONATGE*\t(anchor1) (G) / (anchor2) (G)... text verbal

  Tipus de tokens dins parèntesis:
    - Numèrics:  (21), (47), (48), (52) → àncores temporals (SS del mateix HH:MM del TAKE)
    - Literals:  (G), (GS), (OFF), (ON), (OFF-ON), (PG), (RÍE) → gesticulació/acotació

  Interpretació numèrica: base_tc = 00:06:24
    anchor (47) → minute_start(384) = 360 → 360 + 47 = 407s = 00:06:47

PROBLEMA DETECTAT:
  Per a línies amb MÚLTIPLES anchors numèrics intercalats amb gesticulació:
    *RUFFY*\t(GS) / (28) (G) / (41) (G)... Haki busoshoku! Enduriment!

  Comportament ANTERIOR:
    abs_time = min(anchors) = anchor(28) = 00:06:28  ← incorrecte
    El diàleg "Haki busoshoku!" comença a 00:06:41, no a 00:06:28

  Comportament ESPERAT:
    abs_time = 00:06:41 (l'anchor immediately adjacent al text verbal)

CAUSA:
  _normalize_line_text() retornava has_prefix=False quan no hi ha text verbal
  ABANS del primer anchor. En aquest cas, parse_script_txt_from_text() usava
    abs_time = min(anchors)
  però hauria d'usar l'anchor que precede directament el text verbal.

FIX IMPLEMENTAT:

  _normalize_line_text() ara rastreja last_anchor_before_verbal:
    Per a cada anchor numèric trobat, comprova si hi ha text verbal (ALNUM fora
    de parèntesis) en el raw restant. Si sí, actualitza last_anchor_before_verbal.
    NOTA: els blocs (G),(GS),(OFF) etc. s'eliminen del raw per la comprovació,
    de manera que no compten com a "text verbal".

  Nou valor de retorn (5è): Optional[float] last_anchor_before_verbal

  parse_script_txt_from_text() — nova lògica de càlcul abs_time:
    if has_prefix:
        abs_time = last_anchor_time_in_take   (igual que abans)
    elif last_anchor_before_verbal is not None:
        abs_time = last_anchor_before_verbal  # NOU: últim anchor adjacent al diàleg
    else:
        abs_time = min(anchors_u)             (fallback, igual que abans)

EXEMPLES:

  1. *RUFFY*\t(GS) / (28) (G) / (41) (G)... Haki busoshoku! Enduriment!
     ANTERIOR: abs_time = 00:06:28 (min)
     CORREGIT: abs_time = 00:06:41 (últim anchor avant verbal)

  2. *KRAKEN*\t(23) (G)
     → anchors = [23_based], text verbal = "" → _is_verbal = False → CUE OMÈS ✓
     (comportament INALTERAT, ja funcionava)

  3. *SPEAKER*\t(28) text verbal
     → last_anchor_before_verbal = 28_based (verbal segueix (28))
     → abs_time = 28_based (IGUAL que min = 28_based) ✓

  4. *SPEAKER*\ttext verbal (28) text
     → has_prefix = True (verbal AVANT primer anchor)
     → abs_time = last_anchor_time_in_take (anchor de cue anterior) ✓ INALTERAT

  5. *SPEAKER*\t(28) verbal (41) no_verbal
     → anchor(28): verbal restant? sí ("verbal (41) no_verbal" → "verbal  no_verbal") ✓
     → anchor(41): verbal restant? "no_verbal" → però no... espera:
        "no_verbal" TÉ ALNUM ("no_verbal") → last_anchor_before_verbal = 41!
        Hmm, però aquí el verbal és ENTRE (28) i (41). Llavors last=41 seria incorrecte?
     NOTA: en la pràctica als guions SONILAB, text verbal sol anar DESPRÉS de tots els
     anchors. El cas (28) verbal (41) no_verbal és atípic i poc freqüent. El fallback
     a l'anchor més proper és acceptable per a la majoria de casos reals.

COMPORTAMENT AMB LINES PURAMENT DE GESTICULACIÓ:
  - (G),(GS),(OFF),(ON),(OFF-ON),(PG),(RÍE) → PAREN_HAS_ALPHA_RE match → space en blanc
  - Detectat a _normalize_line_text() abans d'aquest fix (sense canvi)
  - _is_verbal_after_strip() filtra correctament línies sense ALNUM → cues omeses ✓
